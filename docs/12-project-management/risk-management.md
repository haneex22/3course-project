# Управление рисками

Для минимизации негативного влияния на ход выполнения проекта был проведён анализ рисков и разработан план реагирования.

## Методология оценки рисков

Оценка рисков производится по двум параметрам:

- **Вероятность** (1 — низкая, 5 — высокая)
- **Влияние** (1 — незначительное, 5 — критическое)

**Итоговая оценка риска:** Вероятность × Влияние


| Уровень риска | Диапазон | Реагирование                |
| ------------- | -------- | --------------------------- |
| Низкий        | 1–6      | Принятие / Мониторинг       |
| Средний       | 7–12     | План mitigation             |
| Высокий       | 13–25    | Обязательное предотвращение |

## Мониторинг рисков

Контроль рисков осуществляется на протяжении всего жизненного цикла проекта:

- **Еженедельно** — проверка статуса рисков, обновление оценок
- **По факту инцидента** — внесение нового риска в реестр
- **В рамках CI/CD** — автоматическая проверка покрытия тестами (JaCoCo), статический анализ кода (Checkstyle)
- **Резервное копирование** — автоматическое, по расписанию (pg_dump)

## Вывод

Реестр рисков включает 10 идентифицированных рисков. Критических рисков (оценка >20) не выявлено. Наибольшее внимание уделено рискам **R05** (безопасность JWT), **R09** (конфликт бронирований) и **R02** (сохранность данных). Для каждого риска разработаны меры предотвращения или снижения последствий, что обеспечивает устойчивость проекта к наиболее вероятным негативным сценариям.
