# Бизнес-глоссарий

| Термин | Определение |
|--------|-------------|
| **Бронирование (Reservation)** | Предварительный резерв автомобиля на конкретный период; создаётся клиентом и действует в статусе PENDING 15 минут до подтверждения оплатой |
| **Арендатор (Client)** | Физическое лицо, имеющее аккаунт в системе, прошедшее верификацию и заключающее договор аренды |
| **Автопарк** | Совокупность транспортных средств, зарегистрированных в системе и доступных для аренды |
| **Договор аренды (Rental Agreement)** | Юридический документ, фиксирующий факт выдачи автомобиля арендатору; содержит начальный пробег и уровень топлива |
| **Депозит** | Залоговая сумма, удерживаемая при заключении договора для покрытия потенциального ущерба |
| **Тариф (Pricing)** | Стоимость аренды за единицу времени (сутки); может варьироваться в зависимости от дней недели (Strategy) |
| **Статус автомобиля (CarStatus)** | Текущее состояние ТС: AVAILABLE, RESERVED, RENTED, MAINTENANCE |
| **Выдача** | Передача автомобиля арендатору с фиксацией пробега и уровня топлива; переводит бронь в статус ACTIVE |
| **Возврат** | Приём автомобиля от арендатора; переводит бронь в COMPLETED, ТС — в AVAILABLE |
| **Верификация (Verification)** | Проверка подлинности документов клиента (паспорт, водительские права); только верифицированные клиенты могут бронировать |
| **JWT (JSON Web Token)** | Подписанный токен для аутентификации API-запросов; действует 24 часа |
| **PCMEF** | Архитектурный паттерн: Presentation → Control → Mediator → Entity → Foundation; описывает слои системы |
| **REST API** | Программный интерфейс бэкенда; доступен на порту 8080; использует JSON |
| **Оффлайн-режим** | Режим работы Android-клиента без интернет-соединения с использованием данных из Room-кэша |
| **Менеджер проката** | Сотрудник с ролью MANAGER; имеет доступ к управлению статусами ТС и просмотру всего автопарка |
| **Автоотмена (Auto-cancellation)** | Автоматическая отмена бронирований в статусе PENDING старше 15 минут через планировщик (@Scheduled) |
