# Контекстная диаграмма IDEF0 (A-0)

## Описание системы

**Система:** Информационная система проката автомобилей  
**Точка зрения:** Администратор системы  
**Цель:** Автоматизировать процесс проката автомобилей от бронирования до возврата

## Диаграмма A-0

```
                    Правила проката, Тарифы
                           ↓
Клиент ──────→ [ ИС ПРОКАТА АВТОМОБИЛЕЙ ] ──────→ Договор аренды
Менеджер ─────→                           ──────→ Отчёты
                                           ──────→ Уведомления клиентам
                           ↑
              СУБД PostgreSQL, Android SDK, Spring Boot
```

**Входы:**
- Заявка на аренду (от клиента)
- Данные для верификации (паспорт, ВУ)
- Команды управления автопарком (от менеджера)
- Платёжные данные

**Выходы:**
- Подтверждение бронирования
- Договор аренды
- Акт возврата
- Аналитические отчёты

**Управление:**
- Политика аренды (не более 1 активного бронирования)
- Автоотмена PENDING через 15 минут
- Верификация клиента обязательна для бронирования

**Механизмы:**
- Spring Boot 3 (бэкенд)
- PostgreSQL 15 (база данных)
- Android-клиент (Kotlin + Compose)
- Docker (инфраструктура)
