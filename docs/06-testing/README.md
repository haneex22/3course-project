# Этап 6. Тестирование

## Цель этапа

Целью этапа является обеспечение качества разработанной информационной системы посредством модульного и интеграционного тестирования серверной части и мобильного приложения.

В ходе тестирования проверялась корректность работы бизнес-логики, REST API, механизмов безопасности, алгоритмов расчёта стоимости аренды и пользовательских сценариев мобильного приложения.

---

# Ключевые артефакты этапа

- План тестирования;
- Модульные тесты (JUnit 5);
- Интеграционные тесты Spring Boot;
- Android Unit Tests;
- Android Instrumentation Tests.

# 6.1 Отчёт о покрытии кода JaCoCo

Для оценки качества тестирования серверной части проекта использовался инструмент **JaCoCo (Java Code Coverage)**, предназначенный для анализа покрытия программного кода модульными тестами.

JaCoCo автоматически формирует отчёт после выполнения тестов Maven и отображает информацию о покрытии классов, методов, строк и ветвлений.

Отчёт формируется в каталоге:

```text
backend/target/site/jacoco/index.html

```

---

## Настройка JaCoCo

В проекте используется Maven-плагин **jacoco-maven-plugin**, который автоматически подключается к процессу выполнения тестов и генерирует HTML-отчёт после завершения сборки.

Отчёт содержит информацию о покрытии следующих компонентов проекта:

- сервисный слой;
- бизнес-логика бронирования;
- алгоритмы расчёта стоимости аренды;
- обработка состояний бронирования;
- контроллеры;
- механизмы безопасности.
