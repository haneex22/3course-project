# Этап 9. API-документация

## Цель этапа

Целью данного этапа является документирование REST API серверной части информационной системы проката автомобилей. REST API обеспечивает взаимодействие мобильного Android-приложения с серверной частью, реализованной на платформе Spring Boot.

Обмен данными между клиентом и сервером осуществляется по протоколу HTTP с использованием формата JSON.

---

## Базовый URL

```
http://localhost:8080/api/v1

```

---

## Аутентификация

Для защиты ресурсов используется механизм JWT-аутентификации.

После успешной авторизации сервер возвращает JWT-токен, который передается в заголовке каждого защищённого запроса.

```
Authorization: Bearer <JWT_TOKEN>

```

# 9.1 AuthController

Базовый путь:

```
/api/v1/auth

```


| Метод | Endpoint    | Назначение               | Авторизация |
| ----- | ----------- | ------------------------ | ----------- |
| POST  | `/register` | Регистрация пользователя | Нет         |
| POST  | `/login`    | Авторизация пользователя | Нет         |


---

### POST /api/v1/auth/register

Регистрация нового пользователя.

**Тело запроса**

```json
{
  "email": "user@example.com",
  "password": "password123",
  "firstName": "Иван",
  "lastName": "Иванов"
}

```

Ответ:

```
201 Created

```

---

### POST /api/v1/auth/login

Авторизация пользователя.

**Тело запроса**

```json
{
  "email": "user@example.com",
  "password": "password123"
}

```

Ответ

```json
{
    "token":"JWT_TOKEN"
}

```

# 9.2 CarController

Базовый путь

```
/api/v1/cars

```


| Метод | Endpoint            | Назначение                      | Авторизация    |
| ----- | ------------------- | ------------------------------- | -------------- |
| GET   | `/cars`             | Получить список автомобилей     | Нет            |
| GET   | `/cars/{id}`        | Получить сведения об автомобиле | Нет            |
| GET   | `/cars/{id}/busy`   | Получить занятые периоды        | Нет            |
| PUT   | `/cars/{id}/status` | Изменить статус автомобиля      | ADMIN, MANAGER |


---

### GET /api/v1/cars

Возвращает список автомобилей.

Поддерживаемые параметры:


| Параметр  | Тип           |
| --------- | ------------- |
| carClass  | String        |
| startDate | LocalDateTime |
| endDate   | LocalDateTime |


---

### GET /api/v1/cars/{id}

Возвращает информацию об автомобиле по идентификатору.

---

### GET /api/v1/cars/{id}/busy

Возвращает список занятых периодов выбранного автомобиля.

---

### PUT /api/v1/cars/{id}/status

Изменяет текущий статус автомобиля.

Требуется роль **ADMIN** или **MANAGER**.

# 9.3 BookingController

Базовый путь

```
/api/v1/bookings

```


| Метод | Endpoint                | Назначение                        | Авторизация  |
| ----- | ----------------------- | --------------------------------- | ------------ |
| POST  | `/bookings`             | Создать бронирование              | Пользователь |
| GET   | `/bookings/my`          | Получить собственные бронирования | Пользователь |
| GET   | `/bookings/{id}`        | Получить бронирование             | Пользователь |
| POST  | `/bookings/{id}/cancel` | Отменить бронирование             | Пользователь |


---

### POST /api/v1/bookings

Создание нового бронирования автомобиля.

Ответ

```
201 Created

```

---

### GET /api/v1/bookings/my

Возвращает список бронирований текущего пользователя.

---

### GET /api/v1/bookings/{id}

Возвращает сведения о выбранном бронировании.

---

### POST /api/v1/bookings/{id}/cancel

Отменяет бронирование пользователя.

Ответ

```
204 No Content

```

# 9.5 Коды ответов HTTP


| Код                       | Значение                                |
| ------------------------- | --------------------------------------- |
| 200 OK                    | Запрос успешно выполнен                 |
| 201 Created               | Ресурс успешно создан                   |
| 204 No Content            | Запрос выполнен без возвращаемых данных |
| 400 Bad Request           | Ошибка входных данных                   |
| 401 Unauthorized          | Пользователь не авторизован             |
| 403 Forbidden             | Недостаточно прав доступа               |
| 404 Not Found             | Запрашиваемый ресурс не найден          |
| 500 Internal Server Error | Внутренняя ошибка сервера               |


---

# 9.6 Используемые технологии

В рамках реализации REST API используются следующие технологии:

- Spring Boot;
- Spring Web;
- Spring Security;
- JWT-аутентификация;
- Spring Data JPA;
- PostgreSQL;
- Hibernate Validator;
- Jackson JSON;
- Retrofit (Android-клиент).

---

# Ключевые артефакты этапа

В результате выполнения данного этапа разработаны:

- REST API серверной части;
- контроллеры AuthController, CarController, BookingController и AdminController;
- DTO для запросов и ответов;
- JWT-аутентификация пользователей;
- REST-клиент Android-приложения на основе Retrofit.
