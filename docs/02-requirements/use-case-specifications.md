# Спецификации вариантов использования

## UC-003: Забронировать автомобиль

| Поле | Значение |
|------|----------|
| **Идентификатор** | UC-003 |
| **Название** | Забронировать автомобиль |
| **Актор** | Клиент |
| **Предусловия** | Клиент аутентифицирован; аккаунт верифицирован (`isVerified = true`); нет активных бронирований |
| **Постусловия** | Создано бронирование в статусе CONFIRMED; автомобиль переведён в RESERVED; отправлено уведомление |
| **Приоритет** | Высокий |

### Основной поток

| № | Действие актора | Реакция системы |
|---|-----------------|-----------------|
| 1 | Открывает карточку автомобиля | Отображает информацию об автомобиле и кнопку "Забронировать" |
| 2 | Нажимает "Забронировать" | Переходит на экран BookingScreen |
| 3 | Выбирает даты начала и окончания аренды | Рассчитывает и отображает предварительную стоимость (Strategy) |
| 4 | Нажимает "Подтвердить бронирование" | Вызывает `BookingFacade.processCompleteBooking()` |
| 5 | — | Проверяет верификацию клиента |
| 6 | — | Проверяет отсутствие пересечений дат (ReservationRepository) |
| 7 | — | Создаёт бронирование в статусе PENDING |
| 8 | — | Симулирует оплату (PaymentServiceImpl) |
| 9 | — | Переводит бронирование в CONFIRMED (State: PendingState → ConfirmedState) |
| 10 | — | Асинхронно отправляет уведомление о подтверждении |
| 11 | Видит экран профиля с новым бронированием | — |

### Потоки исключений

| Условие | Действие системы |
|---------|-----------------|
| Клиент не верифицирован | Возвращает HTTP 403, сообщение "Необходима верификация документов" |
| Даты пересекаются с другим бронированием | Возвращает HTTP 409, сообщение "Автомобиль недоступен на выбранные даты" |
| Автомобиль не в статусе AVAILABLE | Возвращает HTTP 409, сообщение "Автомобиль недоступен" |
| Период ≤ 0 | Валидация на клиенте, кнопка неактивна |

## UC-007: Осмотреть и выдать автомобиль

| Поле | Значение |
|------|----------|
| **Идентификатор** | UC-007 |
| **Название** | Осмотреть и выдать автомобиль |
| **Актор** | Менеджер проката |
| **Предусловия** | Бронирование в статусе CONFIRMED; клиент присутствует; менеджер аутентифицирован с ролью MANAGER/ADMIN |
| **Постусловия** | Создан договор аренды; бронирование переведено в ACTIVE; автомобиль переведён в RENTED |
| **Приоритет** | Высокий |

### Основной поток

| № | Действие актора | Реакция системы |
|---|-----------------|-----------------|
| 1 | Открывает детали бронирования | Отображает данные клиента и автомобиля |
| 2 | Проверяет документы клиента (паспорт, ВУ) | — |
| 3 | Осматривает автомобиль, фиксирует пробег и уровень топлива | — |
| 4 | Нажимает "Выдать автомобиль" | Вызывает `ConfirmedState.handoverCar()` |
| 5 | — | Создаёт запись `RentalAgreement` с начальными параметрами |
| 6 | — | Переводит бронирование в статус ACTIVE |
| 7 | — | Переводит автомобиль в статус RENTED |
| 8 | Печатает договор и передаёт клиенту ключи | Отображает подтверждение |

### Потоки исключений

| Условие | Действие системы |
|---------|-----------------|
| Бронирование не в статусе CONFIRMED | Возвращает ошибку "Некорректный статус бронирования" |
| Клиент без верификации | Запрет — `isVerified` проверяется при создании брони |
| Автомобиль на техобслуживании | Менеджер переводит статус ТС в AVAILABLE вручную, затем создаёт договор |
