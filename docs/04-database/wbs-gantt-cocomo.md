## 3. COCOMO — Оценка трудоёмкости

### 3.1. Исходные данные

**Метод:** COCOMO Basic (Органический тип проекта)

**Параметры:**
- Оценка размера кода (LOC):

| Компонент | Язык | LOC (приблизительно) |
|-----------|:----:|:--------------------:|
| Backend (Java) | Java | ~3 500 |
| Android (Kotlin) | Kotlin | ~4 000 |
| Тесты | Java/Kotlin | ~1 000 |
| **Итого** | | **~8 500 LOC** |

### 3.2. Расчёт (Basic COCOMO)

**Формула:** PM = A × (KDSI)^B

Для органического типа:
- A = 2.4
- B = 1.05
- KDSI = 8.5 (тысяч строк кода)

**Трудоёмкость:**
PM = 2.4 × 8.5^1.05 = 2.4 × 9.7 ≈ 23.3 человеко-месяца

**Время разработки (TDEV):**
TDEV = 2.5 × PM^0.38 = 2.5 × 23.3^0.38 = 2.5 × 3.4 ≈ 8.5 месяцев

**Средняя численность команды:**
SS = PM / TDEV = 23.3 / 8.5 ≈ 2.7 человека

### 3.3. Уточнение (Intermediate COCOMO)

С учётом факторов стоимости (драйверов):

| Фактор | Описание | Рейтинг | Значение |
|--------|----------|:-------:|:--------:|
| RELY | Надёжность ПО | Номинальный | 1.00 |
| DATA | Размер БД | Высокий | 1.08 |
| CPLX | Сложность | Номинальный | 1.00 |
| TIME | Ограничения быстродействия | Номинальный | 1.00 |
| STOR | Ограничения памяти | Низкий | 0.87 |
| VIRT | Изменчивость платформы | Номинальный | 1.00 |
| TURN | Время реакции | Номинальный | 1.00 |
| ACAP | Способности аналитика | Высокий | 0.86 |
| AEXP | Опыт разработки | Номинальный | 1.00 |
| PCAP | Способности программиста | Высокий | 0.86 |
| VEXP | Опыт работы с платформой | Номинальный | 1.00 |
| LEXP | Опыт работы с языком | Номинальный | 1.00 |
| MODP | Использование современных практик | Очень высокий | 0.82 |
| TOOL | Использование инструментов | Очень высокий | 0.78 |
| SCED | График работ | Номинальный | 1.00 |

**EAF =** 1.00 × 1.08 × 1.00 × 1.00 × 0.87 × 1.00 × 1.00 × 0.86 × 1.00 × 0.86 × 1.00 × 1.00 × 0.82 × 0.78 × 1.00 = **0.37**

**Уточнённая трудоёмкость:**
PM_adj = 23.3 × 0.37 ≈ **8.6 человеко-месяца**

**Уточнённое время:**
TDEV_adj = 2.5 × 8.6^0.38 ≈ **5.8 месяцев**
